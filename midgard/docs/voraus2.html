postgreSQL 7.1 (LINUX/UNIX)

Zu Zeit wird die Postgres-Datenbank noch benötigt, nach der Umstellung auf xml wird dies voraussichtlich nicht mehr der Fall sein
Die aktuell(st)en Pakete für postgreSQL sind unter http:/www.postgresql.org/ zu finden.
SuSE (einschließlich SuSE 7.2) hat eigene Postgres Pakete der Version 7.0.3. Diese müssen aktualisiert werden. Von Hause aus installiert SuSE PostgreSQL unter /usr/lib/pgsql, wohingegen /usr/local/pgsql Standard ist.
Bei einem Update ist also darauf zu achten, daß entweder die alten SuSE-Pakete vorher deinstalliert werden oder beim compilieren /usr/lib/pgsql als Installationsverzeichnis (für die Include-Dateien /usr/lib/pgsql/include/) angegeben wird (siehe postgresql/configure --help und INSTALL bzw. README. Weitere Informationen zu Postgres finden sich in der Dokumentation zu Postgres (im Verzeichnis doc)).

Nach der Installation von Postgres:

  1. Erstellen der Datenbank (erster Start von postgres):
    als user 'postgres' (als root 'su postgres' eingeben) muß der 'initdb' eingeben werden (steht auch im INSTALL).

  2. Starten des Datenbank-Servers:
    Der Datenbankserver muß gestartet werden. Damit er automatisch beim booten gestartet werden kann sollte man die folgende Zeile in die Datei /etc/init.d/boot.local einfügen:

    su postgres -c "/usr/local/pgsql/bin/pg_ctl -D /usr/local/pgsql/data -l /var/log/psql.log start"

    Damit das fehlerfrei funktioniert muß die Datei '/var/log/psql.log' natürlich dem Benutzer postgres gehören. (als root:

    'touch /var/log/psql.log'

    und danach:

    'chown postgres /var/log/psql.log'

    Dann kann man (als root) durch 'source /etc/init.d/boot.local' testen ob es korrekt funktioniert.

  3. Anpassen der Pfades zu den Ausführbaren Dateien:
    Die Zeile 

    'export PATH=$PATH:/pfad/zu/pgsql/bin'

    (z.B.: /usr/local/pgsql/bin, /usr/lib/pgsql/bin oder /opt/pgsql/bin) einfügen oder ergänzen (und neu einloggen)

  4. Anlegen eines neuen Benutzers (der Datenbank):
    Als User 'postgres' 

    'createuser NAME'

    'NAME' sollte hierbei der Name sein, mit dem man sich ins System einloggt.
  5. Anlegen der Datenbank midgard:
    als user 'NAME':

    'createdb midgard'

  6. Verfügbar machen der Postgres-Bibliotheken (als root):
    In /etc/ld.so.conf folgende Zeile eintragen:

    '/pfad/zu/pgsql/lib'

    (z.B.: /usr/local/pgsql/bin, /usr/lib/pgsql/bin oder /opt/pgsql/bin)
    Anschließend 'ldconfig' aufrufen.

Gtk+:

Es wird Version 1.2.8 (oder höher) benötigt.
Testen mit 'gtk-config --version'
Projektseiten:
http://www.fh-heilbron.de/mirrors/ftp.gimp.org/gtk/v1.2/ oder http://www.gtk.org/
Von diesen Seiten werden die beiden Pakete 'glib-1.2.8.tar.gz' und 'gtk+1.2.3.tar.gz' benötigt.
Achtung:
Es kann sein, daß das Paket bei einzelnen Distributionen (z.B. SuSE) in ein Binary- und ein Development-Paket aufgeteilt ist. In diesem Fall muß das Development-Paket zusätzlich nachinstalliert werden. (bei SuSE: gtk-devel, Serie xdev)

Gtk-- / Gtkmm:

Es wird Version 1.2.4 (oder höher) benötigt.
Testen mit 'gtkmm-config --version'
Das Paket heißt je nach Distribution Gtk-- oder Gtkmm.
Projektseiten:
http://gtkmm.sourceforge.net/
Achtung:
Es kann sein, daß das Paket, abhängig von der Distribution in unterschiedliche Verzeichnisse installiert wird. (Kann u.U. zu Problemen beim Kompilieren führen) Es kann sein, daß das Paket bei einzelnen Distributionen (z.B. SuSE) in ein Binary- und ein Development-Paket aufgeteilt ist. In diesem Fall muß das Development-Paket zusätzlich nachinstalliert werden. (bei SuSE: gtkmm-devel, Serie xdev)
